# plan-mode

Use when a task needs explicit decomposition before execution.

## Workflow

1. Restate the goal in one sentence.
2. Break the work into the smallest useful steps.
3. Identify blockers, dependencies, and validation points.
4. Decide what to do immediately and what can wait.
5. Keep the plan short enough to execute without rethinking it each round.

## Rules

- Prefer concrete steps over vague intentions.
- Mark risky or high-uncertainty items clearly.
- Reuse existing code and repo patterns before inventing new ones.
- Do not over-plan tasks that are already straightforward.
- Revisit the plan only when the user changes the goal or scope.
